Three Delivery Paths

Pick a path by team maturity

Command

node src/cli.js . --format html --out reports/scan.html --fail-on p1

Outcome

Catch high-risk blockers before merge and reduce last-minute release rework.

Command

node src/cli.js . --preset release --format sarif --out reports/scan.sarif --fail-on p1

Outcome

Standardize quality gates across repos with the same policy and threshold.

Command

node src/cli.js fleet-scan --discover-root ../workspace --discover-depth 4 --export-repos reports/repos.txt --export-only
node src/cli.js fleet-scan --repos-file reports/repos.txt --history-dir reports/fleet-history --format html --out reports/fleet.html --continue-on-error --fail-on p1

Outcome

Track failed repos, slow repos, and recovery cadence in one operational view.
