Repo Sleep Doctor
Release Risk Scanner + Fleet Analytics

上线前，不靠感觉；
用数据看见仓库风险

Repo Sleep Doctor 面向工程团队的发布守门场景：自动扫描冲突标记、潜在密钥、调试残留、文档与脚本缺口， 并输出单仓报告与跨仓 Fleet 看板，支持缓存加速与趋势跟踪。

查看 GitHub 仓库 阅读中文文档 
# 单仓扫描
node src/cli.js . --format html --out reports/scan.html --cache-file reports/scan.cache.json --fail-on p1

# 多仓扫描 + Fleet 汇总
node src/cli.js fleet-scan ../repo-a ../repo-b --history-dir reports/fleet-history --scan-out-dir reports/fleet-reports --scan-format html --format html --out reports/fleet.html --preset release --cache-dir reports/fleet-cache --fail-on p1

核心能力

发布阻断识别

P0/P1/P2 分级输出风险，优先处理真正影响上线的关键问题，降低“带病发布”概率。

趋势与动量可视化

HTML 报告内置分数趋势、规则动量、热点文件视图，帮助快速定位风险聚集点。

组织级 Fleet 汇总

一条命令扫描多仓、写入历史、聚合看板，形成跨仓库风险排行榜和治理优先级。
